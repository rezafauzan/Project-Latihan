Catatan

        Specificity
        Merupakan Konsep bahwa setiap deklarasi css (selector) memiliki berat yang berbeda, berat ini menentukan seberapa spesifik sebuah elemen dapat dipilih oleh selector

        Contoh kasus : Lihat Pada catatan.docx

        Syntax css diatas akan menjadi ambigu dan menimpa(Overiding) sehingga baris paling bawah yang dipakai 

        Namun ketika selectornya memiliki bobot spesifik sehingga akan terjadi kalkulasi dimana selector yang memiliki bobot paling tinggi akan digunakan 

        Menghitung Specificity
            Anggaplah terdapat 4 digit angka masing masing digit memiliki label seperti pada gambar yaitu inline, id, class, element
            Dan untuk menghitung nilai specificity nya bisa dilihat pada selectornya misal apabila selectornya menggunakan 1 buah class seperti .namaClass maka digit class akan bertambah 1 
            Kembali pada contoh kasus antara selector #p1 dengan p memiliki bobot specificity sebagai berikut 

            #p1 = 0 1 0 0 dan p = 0 0 0 1 maka pada kasus ini #p1 memiliki nilai specificity yang lebih tinggi sehingga mengalahkan selector p

            Contoh kasus lain yang lebih kompleks adalah selector ul#sarapan li dengan .favorit pada gambar diatas

            Hasilnya : lihat pada catatan.docx

            Karena selector ul#sarapan li memiliki nilai specificity = 0 1 0 2 sedangkan selector .favorit = 0 0 1 0 sehingga selector ul#sarapan li lebih spesifik
            Dan jika ingin menampilkan tulisan nasi goreng(elemen ber class .favorit) dengan warna merah maka harus diberikan beban specificity tambahan agar bisa lebih spesifik contoh ul#sarapan li.favorit

            Untuk mempermudah penghitungan terdapat website untuk menghitung beban specificity pada selector css yaitu di http://specificity.keegan.st
